# Literary Context Media Generation

## Концепция

Система "Литературного контекста" (Literary Context, далее LC) позволяет привязать
генерацию всех медиа-материалов карточки (изображение, аудио, подсказка, примеры
предложений) к конкретному литературному произведению или корпусу текстов.

Вместо абстрактной "собаки" пользователь видит сцену из "Хамелеона" Чехова:
полицейский Очумелов, укушенный Хрюкин, испуганная собака у дровяного склада.
Подсказка — "зверь, который укусил мастера", а пример предложения — точная цитата
из рассказа.

---

## Оценка идеи

### Почему это сильно

1. **Эмоциональная привязка.** Запоминание через сюжет и образы многократно
   эффективнее, чем через абстрактные картинки. Это научно подтверждённый принцип
   — метод локусов, мнемонические истории.

2. **Уникальность.** Ни Anki, ни Duolingo, ни Memrise не предлагают ничего
   подобного. Это потенциально killer-feature.

3. **Двойная польза.** Пользователь одновременно учит язык И знакомится с
   литературой на целевом языке.

4. **Переиспользование сцен.** Одна сгенерированная картинка покрывает 5-15 слов
   из той же сцены — экономия на генерации изображений.

5. **Синергия с будущим сервисом чтения.** Хранимые тексты произведений станут
   основой для отдельного модуля "чтение с подсветкой незнакомых слов".

### Сложность

**Оценка: высокая (8/10).** Основные источники сложности:

- RAG-система для точного извлечения цитат из текстов
- Управление корпусом текстов (загрузка, индексация, перевод)
- Интеллектуальный матчинг слов к сценам
- Хранение нескольких версий медиа на одно слово (по одной на каждый контекст)
- Новая интеграция (ElevenLabs)

### Масштабируемость

**Горизонтальная:** каждый новый "контекст" (автор/книга) — это добавление текста
в корпус + индексация. Архитектура не меняется.

**Вертикальная:** сцены и их изображения общие между пользователями И между
языками. Картинка "Хамелеон, сцена 1" генерируется ОДИН раз и используется
всеми — и тем, кто учит немецкий, и тем, кто учит португальский. Это возможно
благодаря модели SceneAnchor, которая отделяет визуальную сцену (языконезависимую)
от текстовых фрагментов (языкозависимых).

**Узкое место:** генерация изображений — самая дорогая операция. Но благодаря
двойному переиспользованию (между словами + между языками), стоимость
амортизируется очень эффективно. Пример: одна картинка "Хамелеон, сцена 1"
покрывает ~8 слов x ~5 языков = ~40 WordContextMedia записей.

### Ресурсоёмкость

| Этап | Разовые затраты | Затраты на слово |
|------|-----------------|------------------|
| Загрузка текста в корпус | ~0 (public domain) | — |
| Индексация + эмбеддинги | ~$0.01-0.05 за рассказ | — |
| Поиск сцены для слова | — | ~$0.001 (embedding search) |
| Генерация промпта сцены | — | ~$0.002 (gpt-4o-mini) |
| Генерация изображения | — | ~$0.04 (DALL-E 3), shared |
| Генерация подсказки | — | ~$0.001 |
| Озвучка (ElevenLabs) | — | ~$0.01-0.03 |
| **Итого на слово** | | **~$0.05-0.08** |

Для сравнения, текущая генерация без контекста: ~$0.04-0.05 на слово.
Удорожание ~50-60%, но с учётом переиспользования сцен — может быть даже дешевле.

## Архитектура

### Новые модели данных

```
LiterarySource
├── id
├── slug (unique)              # "chekhov", "bible", "harry-potter"
├── name                       # "Рассказы А.П. Чехова"
├── name_translations (JSON)   # {"de": "Erzählungen von A.P. Tschechow", ...}
├── author                     # "А.П. Чехов"
├── description (Text)
├── cover_image (ImageField)   # обложка для UI
├── source_language             # "ru" — язык оригинала
├── available_languages (JSON)  # ["de", "en", "fr"] — доступные переводы
├── is_active (bool)
├── created_at
└── updated_at

LiteraryText
├── id
├── source (FK → LiterarySource)
├── title                       # "Хамелеон"
├── title_translations (JSON)   # {"de": "Das Chamäleon", ...}
├── language                    # язык этого конкретного текста
├── original_language_text_id (FK → self, nullable)  # ссылка на оригинал
├── full_text (TextField)
├── is_indexed (bool)           # проиндексирован ли для поиска
├── created_at
└── updated_at

SceneAnchor (языконезависимая "точка" в произведении)
├── id
├── source (FK → LiterarySource)
├── text_slug (CharField)       # "hameleon" — какой рассказ
├── fragment_index (int)        # порядковый номер в тексте
├── scene_description (Text)    # AI-описание сцены НА АНГЛИЙСКОМ (для промпта картинки)
├── characters (JSON)           # ["Ochumelov", "Khryukin"] — латиницей, универсально
├── mood (CharField)            # comedic, dramatic, calm, tense...
├── image_file (ImageField)     # сгенерированная картинка сцены (ОДНА на все языки)
├── image_prompt (TextField)    # промпт, по которому генерировалась картинка
├── is_generated (bool)
└── created_at
    # unique_together: (source, text_slug, fragment_index)

LiteraryFragment (языкозависимый текст фрагмента)
├── id
├── anchor (FK → SceneAnchor)   # к какой сцене относится
├── text (FK → LiteraryText)    # к какому тексту (и языку) относится
├── content (TextField)         # текст фрагмента на конкретном языке
├── embedding (VectorField)     # эмбеддинг для поиска (свой для каждого языка!)
├── key_words (JSON)            # ["Hund", "Platz", "Menge"] — на языке фрагмента
└── created_at
    # unique_together: (anchor, text)

ElevenLabsVoiceAssignment (голос привязан к SceneAnchor, не к языку)
├── id
├── anchor (FK → SceneAnchor)
├── voice (FK → ElevenLabsVoice)
└── created_at

WordContextMedia
├── id
├── word (FK → Word)
├── source (FK → LiterarySource)
├── anchor (FK → SceneAnchor, nullable)  # визуальная сцена (image берётся отсюда)
├── fragment (FK → LiteraryFragment, nullable) # конкретный текстовый фрагмент
├── hint_text (TextField)                # подсказка в контексте произведения
├── hint_audio (FileField)               # озвучка подсказки
├── sentences (JSONField)                # предложения из произведения
├── audio_file (FileField)               # озвучка слова голосом контекста
├── is_fallback (bool)                   # True = слово не найдено в произведении
├── created_at
└── updated_at
    # unique_together: (word, source)
```

### Связь с существующими моделями

```
Word (существующая)
├── image_file          # остаётся — "синтетическая" картинка (дефолт)
├── audio_file          # остаётся — "синтетическое" аудио (дефолт)
├── hint_text           # остаётся — "синтетическая" подсказка (дефолт)
├── hint_audio          # остаётся
├── sentences           # остаётся — "синтетические" предложения (дефолт)
├── etymology           # остаётся — НЕ меняется при смене контекста
│
├── context_media → [WordContextMedia]   # reverse FK: все варианты контекстных медиа
│
└── active_context_media()               # метод: возвращает медиа текущего контекста
                                         #         или None (= синтетический режим)

User (существующая, расширяется)
├── active_literary_source (FK → LiterarySource, nullable)
│                          # текущий выбранный контекст, null = синтетический режим
```

### Как это работает: переключение контекста

```
User.active_literary_source = None
  → Word отдаёт свои "родные" поля (image_file, hint_text, sentences)
  → Это текущее поведение, ничего не ломается

User.active_literary_source = "chekhov"
  → Word.active_context_media() ищет WordContextMedia(word=self, source="chekhov")
  → Если найден: отдаёт scene.image_file, hint_text, sentences из контекста
  → Если не найден: отдаёт "родные" поля (fallback)
```

**Переключение мгновенное** — это просто смена FK на User. Все медиа уже
сгенерированы и сохранены в WordContextMedia.

## Пайплайн генерации контекстных медиа

### Этап 0: Подготовка корпуса (делается один раз при добавлении источника)

```
1. Загрузить тексты → LiteraryText (оригинал + переводы)
2. Разбить на фрагменты → LiteraryFragment (по 2-5 предложений)
3. Сгенерировать эмбеддинги для каждого фрагмента
4. Извлечь ключевые слова и персонажей из каждого фрагмента (gpt-4o-mini)
5. Сгенерировать scene_description для каждого фрагмента (gpt-4o-mini)
```

### Этап 1: Матчинг слова к фрагменту (трёхступенчатый)

```
Вход: word="Hund" (собака), source="chekhov", language="de"

Ступень A — точное совпадение (бесплатно, мгновенно):
  1. Получить все LiteraryFragment для source, язык = language
  2. Поиск: "Hund" in fragment.key_words (точное, без учёта регистра)
  3. Найдено? → готово, берём этот фрагмент

Ступень B — семантический поиск (дёшево, быстро):
  1. Получить embedding для слова "Hund"
  2. Cosine similarity с embeddings всех фрагментов (pgvector)
  3. similarity > threshold (0.7)? → берём лучший фрагмент

Ступень C — LLM-матчинг (дорого, но точно):
  Для сложных случаев, когда A и B не дали результата или дали низкий score.
  1. Отправить запрос мощной модели (GPT-5.3 / o3):
     "В каком из этих фрагментов наиболее уместно слово {word}?"
     + top-10 кандидатов из ступени B
  2. LLM выбирает лучший фрагмент или отвечает "ни один не подходит"
  3. "Ни один" → is_fallback=true, стандартная генерация

Результат кэшируется: повторный матчинг того же слова не нужен.
```

### Этап 2: Генерация картинки сцены (если ещё не сгенерирована)

```
Вход: LiteraryFragment → его SceneAnchor

1. Проверить: есть ли уже image_file у SceneAnchor?
2. Если нет:
   a. Взять scene_description из SceneAnchor (на английском — универсальный промпт)
   b. Сгенерировать промпт для DALL-E / Gemini
   c. Сгенерировать изображение → сохранить в SceneAnchor.image_file
   d. Подобрать голос ElevenLabs (по characters/mood) → ElevenLabsVoiceAssignment
3. Если да: переиспользовать. Картинка НЕ зависит от языка пользователя.
   Пользователь учит DE — видит ту же картинку, что и пользователь с PT.
```

### Этап 3: Генерация WordContextMedia

```
Вход: Word, LiteraryFragment (языкозависимый), SceneAnchor (языконезависимый)

1. Извлечь из fragment.content предложения, содержащие слово → sentences
2. Если предложений < 1: расширить поиск на соседние фрагменты того же языка
3. Сгенерировать hint_text на основе контекста сцены (gpt-4o-mini):
   "Опиши одним предложением подсказку к слову {word} в контексте: {fragment}"
4. Озвучить hint через ElevenLabs голосом сцены → hint_audio
   (голос берётся из ElevenLabsVoiceAssignment для этого SceneAnchor)
5. Озвучить слово через ElevenLabs тем же голосом → audio_file
6. Сохранить WordContextMedia:
   - scene → SceneAnchor (image_file берётся оттуда — общая для всех языков)
   - hint_text, sentences — языкозависимые (у каждого пользователя свои)
   - audio_file — языкозависимое (немецкое слово ≠ португальское слово)
```

### Этап 4: Batch-генерация (оптимизация)

```
При генерации контекста для колоды/категории:

1. Собрать все слова
2. Для каждого слова найти лучший фрагмент (Этап 1)
3. Сгруппировать слова по фрагментам:
   Fragment "Хамелеон, сцена 1" → [Hund, Polizist, Platz, Menge]
   Fragment "Каштанка, сцена 3" → [Katze, Zirkus, Clown]
4. Для каждой группы: одна картинка на всех (Этап 2)
5. Для каждого слова в группе: свой WordContextMedia (Этап 3)

Экономия: вместо 7 картинок → 2 картинки.
```

## Интеграция ElevenLabs

### Модель голоса

```
ElevenLabsVoice (справочник)
├── id
├── elevenlabs_voice_id (CharField)
├── name                         # "Narrator Male", "Young Woman"
├── gender (CharField)           # male, female, child
├── style (CharField)            # narrator, dramatic, calm, cheerful
├── language_support (JSON)      # ["de", "en", "ru", "fr"]
├── preview_url (URLField)
└── is_active (bool)
```

### Подбор голоса для сцены

```
1. Анализ фрагмента → определение "настроения" и главного персонажа
2. Маппинг:
   - Мужской персонаж → male voice
   - Женский персонаж → female voice
   - Детский персонаж → child voice
   - Нарратив без явного персонажа → narrator voice
3. Голос закрепляется за Scene и переиспользуется для всех слов сцены
```

## Управление контекстами (UI flow)

### Выбор контекста

```
Настройки / Профиль
└── "Стиль обучения"
    ├── [x] Классический (синтетический)     ← дефолт
    ├── [ ] Рассказы Чехова
    └── [ ] Библия
```

Переключение = `PATCH /api/users/me/ {active_literary_source: "chekhov"}`

### Генерация медиа

```
На странице слова / колоды:

Кнопка "Сгенерировать в стиле Чехова" (если контекст = chekhov)
  → Batch-генерация для выбранных слов
  → Прогресс-бар: "Обработано 12/50 слов..."
  → По завершении: карточки мгновенно переключаются на новые медиа

Кнопка "Сгенерировать классические медиа" (всегда доступна)
  → Текущая генерация без изменений
```

### Смена контекста (мгновенная)

```
Пользователь переключает контекст:
1. active_literary_source меняется
2. API возвращает медиа из WordContextMedia (если есть)
3. Если WordContextMedia нет → fallback на "родные" поля Word
4. Никакой генерации не происходит — всё уже сохранено
```

## Хранение текстов: стратегия наполнения

### Этап 1 (MVP): два источника

| Источник | Язык оригинала | Доступные переводы | Объём |
|----------|----------------|-------------------|-------|
| Рассказы Чехова | ru | de, en | ~50 рассказов |
| Библия | многоязычная | de, en, ru, fr, es, it, pt, tr | полный текст |

**Чехов:** тексты из lib.ru / Project Gutenberg (public domain).
Переводы: существующие немецкие/английские переводы (Gutenberg.org).

**Библия:** многоязычные параллельные тексты свободно доступны
(например, bible-api.com, опенсорсные корпуса).

### Этап 2: расширение

- Братья Гримм (оригинал на немецком — идеально для изучающих DE)
- Андерсен
- Басни Эзопа
- Шекспир (для EN)
- И т.д.

### Формат загрузки

```
Management command:
  python manage.py load_literary_source \
    --slug chekhov \
    --name "Рассказы А.П. Чехова" \
    --texts-dir ./data/chekhov/ \
    --language ru

Структура ./data/chekhov/:
  hameleon.txt          # оригинал
  hameleon.de.txt       # немецкий перевод
  hameleon.en.txt       # английский перевод
  kashtanka.txt
  kashtanka.de.txt
  ...
```

## Порядок изучения по нарративу (опционально)

Если пользователь включает режим "читать по порядку":

```
1. Система берёт первый рассказ → первый фрагмент
2. Извлекает из него незнакомые слова (те, что есть в колоде пользователя)
3. Предлагает учить сначала их
4. После освоения → следующий фрагмент
5. Пользователь "читает" Чехова, попутно учя слова
```

Это опциональный режим. По умолчанию — слова в произвольном порядке,
каждое привязано к наиболее подходящей сцене.

## Хранение нескольких версий медиа: схема данных

### Кросс-языковое переиспользование картинок

```
SceneAnchor (chekhov / hameleon / fragment_index=1)
├── scene_description: "A town square. A frightened dog on three legs..."
├── image_file: "scenes/chekhov_hameleon_01.jpg"    ← ОДНА картинка на ВСЕ языки
├── characters: ["Ochumelov", "Khryukin"]
├── mood: "comedic"
│
├── LiteraryFragment (language=de, text="Das Chamäleon")
│   ├── content: "Aus dem Holzlager des Kaufmanns Pitschugin..."
│   ├── key_words: ["Hund", "Polizist", "Platz"]
│   └── embedding: [0.12, -0.34, ...]
│
├── LiteraryFragment (language=pt, text="O Camaleão")
│   ├── content: "Do depósito de madeira do comerciante Pítchugin..."
│   ├── key_words: ["cão", "policial", "praça"]
│   └── embedding: [0.15, -0.31, ...]
│
└── LiteraryFragment (language=en, text="The Chameleon")
    ├── content: "From the timber yard of merchant Pichugin..."
    ├── key_words: ["dog", "police", "square"]
    └── embedding: [0.11, -0.36, ...]
```

### Пример: два пользователя, разные языки, одна картинка

```
Пользователь A (учит DE, родной RU):
Word "Hund" (собака)
├── image_file: "images/hund_synthetic.jpg"          ← синтетическая (дефолт)
├── WordContextMedia (source=chekhov)
│   ├── anchor → SceneAnchor (hameleon, fragment 1)
│   │            └── image_file: "scenes/chekhov_hameleon_01.jpg"  ← SHARED!
│   ├── fragment → LiteraryFragment (language=de)
│   ├── audio_file: "audio/hund_chekhov.mp3"         ← немецкое произношение
│   ├── hint_text: "Das Tier, das den Meister Chrjukin gebissen hat"
│   ├── sentences: [{"text": "...läuft ein Hund...", "source": "chekhov"}]
│   └── is_fallback: false

Пользователь B (учит PT, родной RU):
Word "cão" (собака)
├── image_file: "images/cao_synthetic.jpg"           ← синтетическая (дефолт)
├── WordContextMedia (source=chekhov)
│   ├── anchor → SceneAnchor (hameleon, fragment 1)
│   │            └── image_file: "scenes/chekhov_hameleon_01.jpg"  ← TA ЖЕ картинка!
│   ├── fragment → LiteraryFragment (language=pt)
│   ├── audio_file: "audio/cao_chekhov.mp3"          ← португальское произношение
│   ├── hint_text: "O animal que mordeu o mestre Khryukin"
│   ├── sentences: [{"text": "...corre um cão...", "source": "chekhov"}]
│   └── is_fallback: false

Картинка сгенерирована ОДИН раз. Аудио и тексты — свои для каждого языка.
```

### Пример: fallback (слово не найдено)

```
Word "Computer" (компьютер) — нет у Чехова
├── WordContextMedia (source=chekhov)
│   ├── anchor: null
│   ├── fragment: null
│   ├── hint_text: стандартная подсказка
│   ├── sentences: стандартные предложения
│   └── is_fallback: true
```

## API: изменения

### Получение слова (с учётом контекста)

```
GET /api/words/{id}/

Если user.active_literary_source = null:
  → возвращает стандартные поля Word

Если user.active_literary_source = "chekhov":
  → ищет WordContextMedia(word=id, source="chekhov")
  → если найден и is_fallback=false:
      image_file = context_media.scene.image_file
      audio_file = context_media.audio_file
      hint_text = context_media.hint_text
      hint_audio = context_media.hint_audio
      sentences = context_media.sentences
  → если не найден или is_fallback=true:
      возвращает стандартные поля Word

Дополнительно в ответе:
  "context": {
    "source": "chekhov",
    "scene_title": "Хамелеон, сцена 1",
    "is_fallback": false
  }
```

### Генерация контекстных медиа

```
POST /api/words/generate-context-media/
Body: {
  "word_ids": [1, 2, 3, ...],     # или пусто = вся колода
  "source_slug": "chekhov"
}

Response: {
  "task_id": "abc-123",           # async task (Celery)
  "estimated_tokens": 45
}

GET /api/tasks/{task_id}/
  → {"status": "processing", "progress": 12, "total": 50}
```

### Переключение контекста

```
PATCH /api/users/me/
Body: {"active_literary_source": "chekhov"}   # или null для классического

→ Мгновенно. Никакой генерации.
```

## Открытые вопросы (с ответами)

1. **Переводы Чехова.** Берём все доступные переводы в public domain.
   Если какого-то перевода нет — можно сгенерировать через LLM как fallback,
   но с пометкой "машинный перевод".

2. **Качество матчинга.** Двухступенчатый подход:
   - Быстрый проход: точное/морфологическое совпадение по key_words (дёшево)
   - Сложные случаи (Polizist ↔ Polizeiaufseher): отправляем на мощную модель
     (GPT-5.3 / o3 или аналог) с контекстом фрагмента. Дороже, но точнее.
     Эта операция разовая — результат кэшируется в WordContextMedia.

3. **Celery.** Решим при реализации. Возможно достаточно Django background tasks
   или простой очереди. Масштаб небольшой: 2-3K слов на пользователя.

4. **Векторная БД.** Начинаем с pgvector (расширение PostgreSQL). Масштаб
   позволяет: ~2-3K слов на пользователя, 70-80% слов пересекаются между языками
   (базовая лексика универсальна). Это значит, что для SceneAnchor-ов реальный
   объём ещё меньше — одни и те же сцены покрывают "собаку" и на DE, и на PT.
   pgvector легко справится с таким объёмом.

5. **Размер фрагмента.** По предложениям (1-3 предложения на фрагмент).
   Зависит от источника: у Чехова предложения длинные и описательные — 1-2
   достаточно. У Библии стихи короткие — можно 2-3. Настраивается при загрузке
   через параметры management command.

6. **Кэширование сцен между пользователями.** РЕШЕНО: модель SceneAnchor
   хранит картинку отдельно от языкозависимых данных. Картинка генерируется
   один раз и переиспользуется между пользователями И между языками.
   Персонализация сохраняется через языкозависимые hint_text, sentences, audio.

## Оценка масштаба и экономики

### Пересечение слов между языками

Базовая лексика (2000-3000 слов) на 70-80% совпадает между европейскими языками.
"Собака", "дом", "бежать", "красный" — это универсальные концепции.

```
Уникальных концептов (слов) в системе: ~3000
Языков: ~8
SceneAnchor-ов для Чехова (~50 рассказов, ~20 фрагментов каждый): ~1000
SceneAnchor-ов для Библии (~1000 глав, ~5 фрагментов каждый): ~5000

Итого SceneAnchor-ов: ~6000
Картинок для генерации: ~6000 (по одной на SceneAnchor)
Но! Не все SceneAnchor-ы будут использованы — только те, к которым привязаны слова.

Реалистичная оценка используемых SceneAnchor-ов:
  ~3000 слов / ~8 слов на сцену = ~375 уникальных сцен
  ~375 картинок x $0.04 (DALL-E 3) = ~$15 на ВСЮ библиотеку картинок
  Эти 375 картинок обслуживают ВСЕХ пользователей на ВСЕХ языках.
```

### Стоимость на пользователя

```
Новый пользователь включает "Чехова":
- Картинки: $0 (уже сгенерированы для предыдущих пользователей)
- Матчинг слов к сценам: ~$0.005 (эмбеддинг + поиск) x 500 слов = ~$2.50
- Подсказки + предложения: ~$0.002 x 500 слов = ~$1.00
- Аудио (ElevenLabs): ~$0.02 x 500 слов = ~$10.00
- Итого: ~$13.50 на 500 слов

Для сравнения, текущая "синтетическая" генерация 500 слов:
- Картинки: $0.04 x 500 = $20.00
- Аудио (OpenAI TTS): ~$0.01 x 500 = $5.00
- Итого: ~$25.00

Литературный контекст ДЕШЕВЛЕ за счёт переиспользования картинок!
(при условии, что ElevenLabs сопоставим по цене с OpenAI TTS)
```
